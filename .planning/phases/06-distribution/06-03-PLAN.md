**2. Overview (3-4 sentences)**
- What it does: Integrates Kimi K2.5 as a research subagent for Claude Code
- Division of labor: Kimi reads/analyzes, Claude implements
- Key benefit: Large context analysis without context pressure on Claude

**3. Quick Start**
```bash
# Clone and install
git clone https://github.com/username/multi-agent-workflow.git
cd multi-agent-workflow
./install.sh

# Test it works
./skills/kimi.agent.wrapper.sh -r reviewer "Review this codebase"
```

**4. Installation**
- Prerequisites: Kimi CLI ≥1.7.0, bash
- Install options: --global (default), --local, --target
- Windows: mention kimi.ps1 PowerShell shim
- Upgrading: install.sh detects and backs up existing
- Uninstalling: ./uninstall.sh

**5. Usage**

Basic invocation:
```bash
./skills/kimi.agent.wrapper.sh -r <role> "prompt"
```

With template:
```bash
./skills/kimi.agent.wrapper.sh -t <template> "prompt"
```

With diff:
```bash
./skills/kimi.agent.wrapper.sh -t verify --diff "verify changes"
```

**6. Agent Roles**

| Role | Type | Description |
|------|------|-------------|
| reviewer | Analysis (read-only) | Code review, patterns, quality |
| auditor | Analysis (read-only) | Architecture, best practices |
| security | Analysis (read-only) | Vulnerabilities, OWASP |
| debugger | Action (full access) | Bug tracing, root cause |
| refactorer | Action (full access) | Restructure, improve patterns |
| implementer | Action (full access) | Build new features |
| simplifier | Action (full access) | Reduce complexity |

Explain analysis vs action roles.

- "role not found": Check .kimi/agents/ directory
- "template not found": Check .kimi/templates/ directory
- Windows issues: Use kimi.ps1, ensure Git Bash installed
